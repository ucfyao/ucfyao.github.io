面向对象的基础一

发表于 2015-09-01 | 更新于 2019-04-19 | 分类于 积累________原理 | 评论数：
本文字数： 901 | 阅读时长 ≈ 1 分钟

如果希望一步步将面向对象的思想，设计模式熟练掌握。

以下基础知识可能需要常常遍历，直到滚瓜烂熟。

对象是一个自包含的实体，用一组可识别的特性和行为来表示。将一组事物的共性抽离出来。

类就是具有相同的属性和功能的对象的抽象的集合。

一个猫的类。有一个方法返回叫声。

class 是表示定义类的关键字，Cat 就是类的名称，Shout就是类的方法。

注意点（通常的规范，具体请遵从公司规定）：

第一，类名首字母大写，多个单词则各个首字母大写。

第二，对外公开的方法需要用public修饰符。

怎么用这个类呢？很简单，实例化一下就可以了。

实例，就是一个真实的对象，而实例化就是创建对象的过程，使用new关键字来创建。说白了，你就当一个new出来的是个新的存在的class就完事了。

着重提出构造方法，其他的魔术方法都差不多的意思，自行脑补。

构造方法，又叫构造函数，其实就是对类进行初始化。如果实在php5版本以后使用魔术方法__construct(),4以前的版本构造方法与类同名。无返回值，在new的时候调用。也就是说在new一个类的时候默认就会执行这个类中的构造方法。

在实例化的时候，传入一个名字，构造函数自动执行，将名字赋值给了私有变量。然后我们调用叫的方法，返回私有变量$name，你会发现返回的是赋值过后的$name。

习惯性开篇总结

发表于 2015-08-26 | 更新于 2019-04-19 | 分类于 日常________经验分享 | 评论数：
本文字数： 528 | 阅读时长 ≈ 1 分钟

终于抽出一点时间将博客站重新上线。

为了节省时间采用了wordpress框架。对于程序，现在认为有可用即可用，存在即是合理，而不再执着于要用很多时间去自己写一个框架。

高可用，敏捷快速是为建站灵魂。拿来主义，是为了释放时间精力做更加擅长的事情。

在此分享自走上程序之路的一些思维方式，期望新加入程序大军的猿们可以有一个模糊的方向，更快的突破限制，走得更远更快。

常认为能从零到有，从小白到建站技术都略懂一些能在短短时间如此大的跨步，得益于老大的指点：“代码就是工具，是让你来使用的，不要执着于什么语法，什么语言，仅仅是工具而已。但是它可以帮助你完成梦想。”。

写代码和日常生活可能是一样的，师父领进门修行在个人，没有什么捷径，唯有勤学苦练。所以在纠结于要不要去培训班的同学，请先明确自己到底想付出什么，又想要得到什么。

刚刚出学校时，没有任何经验，没有高校背景，找工作确实很难。这时候反省自己有没有努力，有没有决心从零开始在某个行业做下去。考虑好自己所想的，做自己喜欢的，坚持下去。

关于培训的问题。。培训班可能对于一点基础没有的同学也是一个路。但是强烈建议一定要摒弃在培训过程中什么都依赖别人去教的习惯，学会自主解决问题，师父领进门，修行靠个人，切记。
